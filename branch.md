# branch names rules

Branch names help clarify their scope of activity; therefore, you should start with a specific word that generally describes what your branch does. Next to your branch’s name, clearly state what its activities entail

## feat/

features such as craft, item, fonctionnality etc

## bugfix/

fixing bug

## refactor/

refacto architecture

## test/

This type is specifically designed for unit testing and end-to-end testing.

## doc/

write doc

## no-issue/

type that cannot be classified
